# 前端环境配置说明

## 项目框架

本项目使用 **Vite** 作为构建工具（不是 Vue CLI），这是 Vue 3 官方推荐的新一代构建工具。

## 配置文件位置

### 1. Vite 配置文件
- **位置**：`vite.config.js`
- **作用**：Vite 构建配置、代理设置、路径别名等

### 2. API 配置文件
- **位置**：`src/api/index.js`
- **作用**：Axios 实例配置、请求拦截器等

### 3. 环境变量文件（需要创建）
- **位置**：项目根目录下的 `.env` 文件
- **作用**：管理不同环境的配置变量

## 为什么没有 .env 文件？

项目创建时可能没有配置环境变量，而是直接在代码中硬编码了配置值。这是可以工作的，但不够灵活。

## 如何使用 .env 文件

### 1. 创建环境变量文件

在 `frontend` 目录下创建以下文件：

#### `.env`（默认环境）
```env
# API基础路径
VITE_API_BASE_URL=/api

# API代理目标地址（开发环境）
VITE_API_TARGET=http://localhost:8080

# 应用标题
VITE_APP_TITLE=福鼎白茶服务平台
```

#### `.env.development`（开发环境）
```env
VITE_API_BASE_URL=/api
VITE_API_TARGET=http://localhost:8080
VITE_APP_TITLE=福鼎白茶服务平台 - 开发环境
```

#### `.env.production`（生产环境）
```env
VITE_API_BASE_URL=/api
VITE_API_TARGET=https://your-production-domain.com
VITE_APP_TITLE=福鼎白茶服务平台
```

### 2. 在代码中使用环境变量

Vite 的环境变量必须以 `VITE_` 开头，才能在客户端代码中访问。

**在 JavaScript 中使用：**
```javascript
// 使用 import.meta.env 访问
const apiUrl = import.meta.env.VITE_API_BASE_URL
```

**已更新的文件：**
- `src/api/index.js` - 已更新为使用环境变量
- `vite.config.js` - 已更新为使用环境变量

### 3. 环境变量优先级

Vite 会按以下顺序加载环境变量：
1. `.env` - 所有环境都会加载
2. `.env.local` - 本地覆盖（会被 git 忽略）
3. `.env.[mode]` - 特定模式（如 `.env.development`）
4. `.env.[mode].local` - 特定模式的本地覆盖

## 当前配置方式

### 开发环境
- API 地址：通过 Vite 代理 `/api` → `http://localhost:8080`
- 配置位置：`vite.config.js` 中的 `proxy` 配置

### 生产环境
- API 地址：需要配置实际的服务器地址
- 可以通过环境变量或构建时配置

## 注意事项

1. **环境变量命名**：必须以 `VITE_` 开头
2. **Git 忽略**：`.env.local` 等文件已在 `.gitignore` 中，不会被提交
3. **重启服务**：修改 `.env` 文件后需要重启开发服务器
4. **类型安全**：环境变量都是字符串类型

## 推荐配置

建议创建 `.env` 文件来管理配置，这样可以：
- ✅ 区分开发/生产环境
- ✅ 避免硬编码
- ✅ 方便团队协作
- ✅ 保护敏感信息（使用 `.env.local`）

## 快速开始

1. 在 `frontend` 目录下创建 `.env` 文件
2. 添加配置变量（参考上面的示例）
3. 重启开发服务器：`npm run dev`
