# 前端项目启动说明

## 环境要求
- Node.js 16.0 或更高版本（推荐 18+）
- npm 7.0 或更高版本（或使用 yarn、pnpm）

## 启动步骤

### 1. 安装依赖
在 `frontend` 目录下打开终端，执行：
```bash
npm install
```

如果安装速度慢，可以使用国内镜像：
```bash
npm install --registry=https://registry.npmmirror.com
```

或者使用 pnpm（推荐，速度更快）：
```bash
npm install -g pnpm
pnpm install
```

### 2. 启动开发服务器
```bash
npm run dev
```

启动成功后，终端会显示：
```
  VITE v5.x.x  ready in xxx ms

  ➜  Local:   http://localhost:3000/
  ➜  Network: use --host to expose
```

### 3. 访问项目
在浏览器中打开：http://localhost:3000

## 其他命令

### 构建生产版本
```bash
npm run build
```
构建完成后，文件会输出到 `dist` 目录

### 预览生产构建
```bash
npm run preview
```
用于本地预览构建后的生产版本

### 代码检查
```bash
npm run lint
```

## 注意事项

1. **确保后端服务已启动**：前端需要连接后端API（http://localhost:8080/api），请先启动后端服务。

2. **端口冲突**：如果3000端口被占用，Vite会自动尝试下一个可用端口，或者可以在 `vite.config.js` 中修改端口号。

3. **跨域问题**：前端已配置代理，API请求会自动转发到后端。

4. **首次启动**：首次启动可能需要较长时间，因为需要下载所有依赖包。

5. **Vite 优势**：
   - 启动速度极快（秒级启动）
   - 热更新速度快
   - 支持 ES 模块
   - 更好的开发体验

## 常见问题

### 问题1：npm install 失败
**解决方案**：
- 清除npm缓存：`npm cache clean --force`
- 删除 node_modules 文件夹和 package-lock.json，重新安装
- 使用 pnpm：`pnpm install`
- 使用 yarn：`yarn install`

### 问题2：端口被占用
**解决方案**：
- Vite 会自动尝试下一个可用端口
- 或者修改 `vite.config.js` 中的端口号
- 或者关闭占用端口的程序

### 问题3：依赖版本冲突
**解决方案**：
- 删除 `node_modules` 和 `package-lock.json`
- 重新执行 `npm install`

### 问题4：Vite 启动报错
**解决方案**：
- 确保 Node.js 版本 >= 16.0
- 检查 `vite.config.js` 配置是否正确
- 确保 `index.html` 在项目根目录

## 项目结构说明

```
frontend/
├── index.html          # 入口 HTML 文件（Vite 要求放在根目录）
├── vite.config.js      # Vite 配置文件
├── package.json        # 项目依赖配置
├── src/                # 源代码目录
│   ├── main.js        # 应用入口文件
│   ├── App.vue        # 根组件
│   ├── api/           # API 接口
│   ├── components/    # 公共组件
│   ├── router/        # 路由配置
│   ├── store/         # 状态管理
│   └── views/         # 页面组件
└── public/            # 静态资源目录
```

## Vite vs Vue CLI

| 特性 | Vite | Vue CLI |
|------|------|---------|
| 启动速度 | 极快（秒级） | 较慢（分钟级） |
| 热更新 | 极快 | 较慢 |
| 构建工具 | Rollup | Webpack |
| 配置复杂度 | 简单 | 复杂 |
| 现代化 | 支持 ES 模块 | 传统构建 |
